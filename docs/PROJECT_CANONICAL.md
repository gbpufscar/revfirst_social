# RevFirst_Social

Version: 1.0  
Status: Canonical

---

## 1. IDENTIDADE DO PROJETO

Nome: RevFirst_Social  
Tipo: Conversation-Driven Growth Engine  
Owner: RevFirst  
Ambiente: VPS (Hostinger)  
Arquitetura: Multi-Agent System

---

## 2. MISSÃO

RevFirst_Social é um sistema multi-agente projetado para:

Inserir o RevFirst nas conversas certas, no momento certo, com a mensagem certa.

Ele não é um bot de postagem automática.

Ele é um:

Motor de Crescimento Baseado em Conversa (Conversation-Driven Growth Engine)

---

## 3. PRINCÍPIOS FUNDAMENTAIS

### 3.1 Conversation > Broadcast

O sistema prioriza:
- responder conversas existentes
- entrar em threads relevantes
- detectar open calls
- interagir com builders

Antes de:
- postar conteúdo próprio

---

### 3.2 Builder-First Positioning

RevFirst_Social deve sempre posicionar o RevFirst como:
- ferramenta para founders
- focado em receita
- prático
- direto
- anti-hype

---

### 3.3 Qualidade > Volume

Nunca:
- spam
- auto-replies massivos
- respostas genéricas

Sempre:
- contextual
- relevante
- natural

---

### 3.4 Human Override

O sistema deve permitir:
- revisão manual
- aprovação antes de publicar
- modo automático opcional

---

## 4. ESCOPO

### 4.1 Incluído

- Monitoramento do X (Twitter)
- Detecção de:
- Open promotion threads
- Threads virais com builders
- Tendências SaaS
- Geração de replies estratégicos
- Geração de post diário
- Thread hijacking controlado
- Lead watchlist de 7 dias
- Análise de performance
- Otimização contínua
- Integração com Telegram para seed de conteúdo

---

### 4.2 NÃO Incluído

- Auto-DM
- Follow em massa
- Compra de seguidores
- Spam automatizado
- Uso fora das regras do X

---

## 5. ARQUITETURA

### 5.1 Estrutura

RevFirst_Social opera como um sistema multi-agente com:
- Intent Classifier
- Opportunity Scorer
- Reply Writer
- Thread Hijack Agent
- Lead List Agent
- Content Seed Agent
- Brand Consistency Agent
- Anti-Cringe Guard
- Strategy Agent
- Orchestrator

---

### 5.2 Pipeline Base

Ingest -> Classify -> Score -> Generate -> Validate -> Approve -> Publish -> Measure -> Optimize

---

## 6. BRAND VOICE (AUTORIDADE)

RevFirst_Social deve obedecer ao arquivo:

`/config/brand_voice.yaml`

Regras obrigatórias:
- Tom direto
- Builder-first
- Sem hype
- Sem emojis
- Sem buzzwords
- CTA sutil

Se violar -> bloquear publicação.

---

## 7. SEGURANÇA E LIMITES

RevFirst_Social deve obedecer:
- Limite diário de replies
- Limite diário de follows
- Cooldown por thread
- Cooldown por autor
- Monitoramento de taxa de erro

Se detectar comportamento suspeito -> pausar sistema.

---

## 8. TELEGRAM SEEDING

O sistema deve permitir:
- Envio manual de post via Telegram
- Extração de estrutura
- Armazenamento como referência de estilo
- Uso como base para novos posts

Esse mecanismo deve:
- Aprender padrão
- Não copiar texto
- Não replicar integralmente

---

## 9. MÉTRICAS DE SUCESSO

RevFirst_Social será considerado funcional quando:
- 1 post diário consistente
- 5-15 replies estratégicos/dia
- Crescimento orgânico de seguidores
- Aumento de profile visits
- Aumento de conversas com founders

---

## 10. EVOLUÇÃO FUTURA

O sistema deve ser modular para permitir:
- Expansão para LinkedIn
- Expansão para Reddit
- Expansão para IndieHackers
- Dashboard Web
- Multi-conta

---

## 11. PRINCÍPIO DE GOVERNANÇA

Nenhum agente pode:
- Publicar sem passar por:
- Brand Consistency Agent
- Anti-Cringe Guard
- Ignorar limites definidos
- Alterar estratégia sem passar pelo Strategy Agent

---

## 12. DEFINIÇÃO DE PRONTO (V1)

RevFirst_Social V1 é considerado pronto quando:
- Ingest + Classifier + Scoring funcionando
- Reply generation validada
- Telegram seed integrado
- Publicação controlada funcionando
- Relatório semanal automático gerado

---

## 13. REGRA MÁXIMA

RevFirst_Social não existe para postar.

Ele existe para:

Encontrar builders e inserir o RevFirst na conversa certa.
